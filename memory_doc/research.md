# Research

## Project concept

- 개인 포트폴리오 전체를 Grasshopper 스타일의 하나의 큰 노드 캔버스로 구성한다.
- 실제 파라메트릭 계산 엔진은 만들지 않는다. 각 프로젝트는 미리 정의된 입력, 연결 액션, 상태 변화, 결과를 갖는 고정 시나리오로 재생한다.
- 장기적으로 초기 조건 3D → 해체/분석을 상징하는 컴포넌트 → 작업 및 조건 입력 → 결과 3D의 흐름으로 프로젝트를 설명한다.
- 결과 노드에는 향후 Three.js 기반 Rhino 모델 뷰어를 포함할 수 있어야 한다.

## Phase 1 problem definition

Phase 1은 전체 포트폴리오 제작이 아니라 핵심 인터랙션의 기술적 성립 여부를 검증하는 프로토타입이다.

검증 시나리오:

1. 숫자 패널 `5`, `3`, 덧셈을 상징하는 컴포넌트, 결과 패널을 배치한다.
2. 초기에는 `3 → B`만 연결하고 `A`는 비워 둔다.
3. A 입력이 없을 때 컴포넌트는 경고 색상과 경고 표시를 사용한다.
4. 경고 표시에 마우스를 올리면 우측 상단에 `A가 입력되지 않았습니다` 메시지를 표시한다.
5. 사용자가 `5` 패널의 출력 포트에서 A 입력 포트까지 드래그하면 임시 Bézier 와이어가 포인터를 따른다.
6. 유효 포트에 드롭하면 와이어가 연결되고 컴포넌트가 정상 상태로 바뀌며 결과 패널의 표시가 `3`에서 `8`로 바뀐다.
7. 이 변화는 실제 덧셈이 아니라 `missing-input → connected → completed`로 정의된 고정 상태 전이다.

## Feasibility assessment

- 구현 가능성이 높다. 노드 위치와 연결 관계를 데이터로 관리하고, 뷰 변환과 상태 전이를 분리하면 계산 엔진 없이도 요구된 동작을 구현할 수 있다.
- 노드 본체는 HTML 요소, 와이어와 임시 연결선은 같은 월드 좌표계를 공유하는 SVG 레이어로 구성하는 방식이 Phase 1에 적합하다.
- 이동, 확대/축소, 스냅은 모든 노드의 좌표를 직접 다시 쓰기보다 하나의 월드 변환으로 처리하는 편이 일관적이다.
- 와이어는 시작/끝 포트의 월드 좌표로부터 cubic Bézier 경로를 계산할 수 있다. 노드 이동이나 확대/축소 시 같은 좌표 변환을 적용한다.
- 향후 Three.js 뷰어는 노드 내부의 독립 렌더링 영역으로 추가할 수 있다. 노드 캔버스의 카메라와 3D 카메라의 휠/드래그 이벤트 충돌 정책은 별도로 필요하다.

## Proposed state model

- `Node`: 고유 ID, 종류, 월드 좌표, 시각 상태, 입력/출력 포트
- `Connection`: 시작 포트 ID, 끝 포트 ID, 연결 상태
- `Scenario`: 허용된 액션과 그 액션이 유발하는 노드/결과 상태 변경
- `Viewport`: pan x/y, zoom
- `Interaction`: idle, panning, moving-node, connecting

계산값을 유도하지 않고 시나리오의 명시적 전이로 결과를 바꾼다. 예: `connect(panel-5.out, add.A)`가 성공하면 `add.status = normal`, `result.text = 8`.

## Risks and decisions still needed

- 경고 말풍선이 hover 전용이면 터치와 키보드 사용자가 접근할 수 없으므로 focus/click 대체 동작을 고려해야 한다.
- Three.js 노드 위에서 포인터를 조작할 때 3D orbit과 캔버스 pan/zoom 중 어느 쪽이 우선하는지 정의해야 한다.

## Confirmed Phase 1 decisions

- 격자 한 칸은 월드 좌표 기준 가로 15단위, 세로 5단위다.
- 스냅은 격자 스냅이 아니다. 이동 중 인접 컴포넌트의 상·하·좌·우 외곽, 가로·세로 중심, 입력·출력 포트의 수평축에 정렬한다.
- pan은 마우스 우클릭 드래그로 조작한다.
- 확대/축소 범위는 20%에서 300%다.
- 300% 최대 확대에서 월드 1단위가 1 CSS px에 대응한다.
- Phase 1 구현은 HTML 노드와 SVG Bézier 와이어를 하나의 월드 변환 아래 결합하는 구조로 성립했다. 브라우저별 시각 및 포인터 QA는 후속 검증 대상으로 남긴다.
