# Standards

## Canvas

- 캔버스 배경색: `#d4d0c8`
- 캔버스 격자색: `#bbb8b0`
- 배경 그리드의 시각 셀 간격은 월드 좌표 기준 가로 600단위, 세로 200단위다. 이는 초기 15×5단위 기준의 40배다.
- 노드, 포트, 와이어는 동일한 월드 좌표계와 viewport 변환을 사용한다.
- 휠 줌은 포인터 위치를 기준점으로 삼아 그 지점의 월드 좌표가 화면에서 움직이지 않게 한다.

## Components

- 패널 컴포넌트 색: `#fffa5a`
- 컴포넌트 상단 반사 색: `#f0f0f0`
- 컴포넌트 기본 색: `#cfcfcf`
- 경고 컴포넌트 상단 반사 색: `#fed195`
- 경고 컴포넌트 기본 색: `#ff971f`
- 패널에는 줌과 무관하게 화면에서 1 CSS px로 보이는 흰색 약 20% 불투명도의 inner stroke를 사용한다.
- 별도 명시가 없는 크기와 굵기 값은 월드 단위이며 300% 최대 확대에서 CSS px와 1:1로 대응하고 viewport 줌에 따라 함께 스케일된다.
- 프로젝트 제목 Scribble은 캔버스 월드 안에 배치하며 검정 계열의 Black 수준 글꼴 두께를 사용한다.
- 주요 컴포넌트 이름은 흰 배경, 검정 외곽선, 하단 포인터를 가진 Bifocals 스타일 말풍선 라벨로 컴포넌트 위에 표시한다.
- 모델·이미지 뷰어는 제공된 Grasshopper Image Previewer 참조처럼 테두리가 있는 빈 미디어 영역으로 표현하며 실제 미디어 연결 전에는 placeholder 상태를 유지한다.
- 생성 PNG는 네 주요 처리 컴포넌트에만 사용한다. Grasshopper 도구 아이콘의 축소 가독성만 참고하며 단순 도형, 제한된 단색 면, 검정 외곽선만 사용하고 그라데이션·광택·입체 음영은 사용하지 않는다.
- 연결 컴포넌트는 생성 아이콘 없이 이름만 표시한다.
- 연결 컴포넌트는 텍스트 한 줄과 최소 여백에 맞춘 얇은 블록이며 시나리오 데이터의 텍스트 길이로 폭을 결정한다.
- 제작값 패널은 `#fffa5a` 단일 배경색을 사용하고 그라데이션을 사용하지 않는다.
- 주요 컴포넌트 내부는 우측 정렬 입력 라벨, 중앙 아이콘, 좌측 정렬 출력 라벨의 공통 3열 구조를 사용한다. 입출력 라벨은 줄바꿈 없이 가로 한 줄로 표시한다. 본체 크기와 포트 행 위치는 라벨 길이, 아이콘 폭, 열 간격, 좌우·상하 패딩으로부터 시나리오 데이터에서 계산한다.
- Bifocals 이름 말풍선을 제외한 캔버스의 본문 라벨·연결 블록·패널은 Georgia 계열의 동일 서체 체계를 사용한다.
- 주요 컴포넌트의 입출력 라벨, 연결 블록, 제작값 패널, 뷰어 심볼·캡션·메시지는 기존 기준보다 `1.2`배 큰 내부 글자 크기를 사용한다. Bifocals 이름 말풍선, 제목 Scribble, 경고 툴팁, 캔버스 조작 안내는 이 배율에서 제외한다. 글자 배율 변경 시 주요 컴포넌트 열 너비와 연결 블록 폭도 같은 데이터 상수와 안전 여유로 함께 계산한다.
- 경고 표시는 느낌표 원이 아니라 컴포넌트 우측 상단에 붙는 작은 노란 말풍선으로 표현한다.
- 연결 컴포넌트는 무효 데이터 상태에서 주요 컴포넌트와 동일한 주황 경고색을 사용하지만 경고 말풍선은 표시하지 않는다.
- 모델·이미지 뷰어는 시각적 버튼 없이 네 모서리의 투명한 포인터 적중 영역과 리사이즈 커서로 가로와 세로를 독립 조절한다.
- 모든 포트의 보이는 형상은 컴포넌트 경계 뒤에 절반이 가려진 반원이며, 와이어는 경계선에 놓인 원 중심에서 시작·종료한다. 본체 외곽선은 포트보다 위 레이어에 다시 그려져 어떤 배율에서도 끊기지 않아야 한다.
- 확대율이 100%에서 70%로 낮아지는 동안 캔버스 텍스트와 아이콘의 불투명도를 1에서 0으로 선형 보간하며, 70% 미만에서는 완전히 숨긴다. 프로젝트 제목 Scribble은 이 LOD 페이드에서 제외해 모든 줌 단계에서 표시한다. 본체·포트·와이어·뷰어 프레임도 유지한다.

## Interaction

- 포트에서 드래그를 시작하면 임시 cubic Bézier 와이어를 표시한다.
- cubic Bézier 와이어는 더 강한 굴곡을 위해 시작·끝점 차이에 대해 X ratio `0.65`, Y ratio `0.90`을 사용한다. 두 제어점의 X는 시작점에서 65% 지점이며 Y는 시작점에서 각각 10%, 90% 지점이다.
- 임시 연결 와이어의 포인터 쪽 끝에는 진행 방향을 나타내는 삼각 화살표를 표시한다.
- 유효한 대상 포트는 연결 가능 상태를 시각적으로 피드백한다.
- 유효 대상에 드롭하면 연결을 확정하고, 그 외 위치에 드롭하면 임시 와이어를 취소한다.
- 이미 연결된 `5 → A` 상태에서 Ctrl을 누른 채 5 출력 포트에서 A 입력 포트까지 드래그하면 연결을 삭제하고 경고·결과를 초기 상태로 되돌린다.
- 연결 삭제 드래그와 대상 포트는 적색 피드백으로 일반 연결 드래그와 구분한다.
- 연결된 와이어는 양 끝 노드가 이동할 때 즉시 따라간다.
- 노드 스냅은 격자 스냅을 사용하지 않는다.
- 노드 이동 중 인접 컴포넌트의 상·하·좌·우 외곽과 가로·세로 중심에 정렬 스냅을 적용한다.
- 노드 이동 중 컴포넌트의 입력·출력 포트 중심이 같은 수평축에 놓이도록 포트 정렬 스냅을 적용한다.
- 스냅이 성립할 때 해당 정렬축을 가이드로 표시한다.
- 필수 입력이 없는 컴포넌트는 경고 시각 상태와 경고 표시를 사용한다.
- 필수 입력은 단순히 와이어가 연결됐는지가 아니라 해당 와이어의 상류 노드가 유효한 데이터를 출력하는지까지 검사한다. 연결됐지만 상류 데이터가 무효인 입력은 미연결과 같은 경고 상태로 취급한다.
- 프로세서 출력의 유효성은 모든 필수 입력의 유효성에 의존하며, 연결 블록·패널·뷰어처럼 입력을 전달하는 노드는 입력 데이터의 유효성을 후류로 전파한다. 연결 그래프의 순환은 무효 데이터로 판정한다.
- 경고 표시의 hover 또는 focus 시 누락되거나 상류 데이터가 무효인 입력을 설명하는 영어 메시지를 컴포넌트 우측 상단에 표시한다.
- 캔버스 pan은 마우스 우클릭 드래그로 조작한다.
- Alt+좌클릭 수평 드래그는 시작점의 수직 기준선을 중심으로 캔버스를 분리한다. 우측 드래그는 기준선 오른쪽에 완전히 놓인 객체만 우측으로, 좌측 드래그는 왼쪽 객체만 좌측으로 이동하며 기준선을 가로지르는 객체는 움직이지 않는다.
- 캔버스 분리 중 원래 기준선과 현재 경계를 점선으로 표시하고 그 사이에 흰색 반투명 밴드와 드래그 방향 화살표를 표시한다. 분리 가이드는 pan·zoom·월드 높이와 무관하게 viewport의 위에서 아래까지 가득 채운다. 드래그 방향이 바뀌면 처음 위치를 기준으로 반대편 객체 집합으로 즉시 전환한다.
- 확대/축소 범위는 20%에서 300%이며 포인터 위치를 기준으로 휠 줌한다.
- 컴포넌트를 한 번 클릭하면 단일 선택 상태가 된다.
- Shift+클릭은 기존 선택 집합을 유지하면서 대상 객체를 선택에 추가하거나, 이미 선택된 대상이면 선택에서 제외한다. Shift+클릭 자체는 객체 이동을 시작하지 않는다.
- 빈 캔버스를 좌클릭 드래그하면 파란 반투명 Fence를 그리고 교차한 모든 캔버스 객체를 선택한다.
- 다중 선택 집합의 객체 하나를 드래그하면 선택된 모든 객체가 동일 변위로 이동해 상대 위치를 유지한다.
- 다중 선택 집합 내부를 잇는 와이어는 `#86db36` 단색을 사용하고 선택 집합과 외부를 잇는 와이어는 선택 쪽 `#86db36`에서 기존 색으로 전환한다.
- 선택된 컴포넌트의 밝은 면은 `#cdf8a5`, 본체 배경은 `#86db36`으로 표시한다.
- 선택된 컴포넌트에 닿은 와이어는 컴포넌트에서 멀어지는 방향으로 `#86db36`에서 기존 와이어 색으로 전환되는 그라데이션을 사용한다.
- 제목 Scribble도 선택 및 이동 대상이며 선택 시 글자를 짙은 초록색 `#276526`으로 표시한다.
- `Ctrl+Z`는 마지막 편집 액션을 취소하고 `Ctrl+Y`는 취소한 액션을 다시 실행한다. `Ctrl+Shift+Z`도 다시 실행으로 지원한다.
- undo/redo 대상은 노드·다중 선택 그룹 이동, 뷰어 크기 변경, 연결 추가·삭제, Alt 캔버스 분리다. 선택·Fence·pan·zoom은 탐색 상태이므로 편집 히스토리에 기록하지 않는다.
- 연속 포인터 이동으로 구성된 하나의 드래그는 히스토리 한 단계로 기록한다. undo 후 새 편집이 완료되면 기존 redo 기록은 폐기하며 최근 100단계까지만 유지한다.

## Behavior

- 컴포넌트는 범용 계산을 수행하지 않는다.
- 결과 변화는 프로젝트별로 선언된 액션-결과 시나리오를 통해서만 발생한다.
- Phase 1 시나리오에서 `5` 출력과 `A` 입력의 연결 완료는 결과 표시 `8`과 정상 컴포넌트 상태를 만든다.
- 연결 삭제 시 Phase 1 시나리오는 A 입력 누락 경고와 결과 표시 `3`의 초기 상태로 복귀한다.
- 상태 데이터와 렌더링 표현을 분리하여 같은 시나리오가 시각 구현에 종속되지 않도록 한다.
- 활성 연결 상태와 유효 데이터 상태를 분리한다. 와이어는 존재할 수 있지만 무효 데이터를 전달할 수 있으며, 뷰어 준비 상태와 후류 경고는 유효 연결을 기준으로 계산한다.
- 노드, 포트, 초기 연결, 허용 연결과 상태 조건은 프로젝트 시나리오 데이터로 선언하고 범용 캔버스 렌더러가 이를 표시한다.
- Phase 2-1의 `Wood Panel` 한 출력은 Construct 경로와 Fabrication 경로의 두 연결 컴포넌트로 명시적으로 분기하며 각 와이어를 독립 연결로 선언한다.
- Phase 2-1 기본 배치에서 Construct용 `Wood Panel` 분기는 주 연결 블록보다 위에 두고 `CONSTRUCT WOOD PANEL 3D`의 Wood Panel 입력과 같은 수평축에 정렬한다. Fabrication용 분기는 아래에 두어 Fabrication 입력으로 연결하며, Construct 본체와 후속 3D·2D 흐름은 Fabrication보다 위쪽 축에 배치한다.

## Accessibility and resilience

- hover로 제공되는 정보는 키보드 focus로도 확인할 수 있어야 한다.
- 줌 수준과 무관하게 포트의 실제 포인터 적중 영역은 조작 가능한 크기를 유지하도록 검토한다.
- 시각 효과가 비활성화되거나 애니메이션 감소 설정이 켜져도 연결 상태와 경고 의미가 유지되어야 한다.
- 패널의 텍스트는 세로 중앙, 가로 왼쪽에 정렬하고 일정한 안쪽 여백을 둔다.

## Hosting portability

- 애플리케이션의 캔버스, 노드, 시나리오, 3D 및 콘텐츠 계층은 특정 호스팅 제공자의 API를 직접 호출하지 않는다.
- 호스팅별 환경 변수, 저장소, 데이터베이스, 오브젝트 스토리지 및 런타임 API는 별도의 설정 또는 어댑터 경계 뒤에 둔다.
- 정적 자산 경로와 공개 URL은 단일 호스트 이름을 코드에 하드코딩하지 않는다.
- 프로젝트 콘텐츠와 시나리오는 가능한 한 이식 가능한 JSON 또는 TypeScript 데이터 구조로 관리한다.
- 브라우저 전용 기능은 표준 Web API를 우선하며, 제공자 전용 기능을 사용할 때는 대체 구현 또는 이전 경로를 기록한다.
- 배포 환경이 바뀌어도 핵심 인터랙션 테스트와 렌더링 기준을 재사용할 수 있어야 한다.
- ChatGPT Sites용 `.openai/hosting.json`과 배포 설정은 애플리케이션 도메인 로직과 분리한다.
- 프로토타이핑 기간에는 검증을 통과한 구현 수정마다 현재 비공개 ChatGPT Sites 버전을 게시한다. 사용자가 로컬 전용을 요청한 경우에만 게시를 생략한다.
