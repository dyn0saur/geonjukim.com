# Plan

## Phases

### Phase 0 — 요구사항 및 기술 검토

핵심 경험을 상태 기반 노드 캔버스로 구현할 수 있는지 검토하고, 확정 규칙과 미결정을 분리한다.

### Phase 1 — 핵심 인터랙션 프로토타입

`5 + 3 → 8` 고정 시나리오로 이동, Bézier 연결, 경고, 줌, 스냅의 성립 여부를 검증한다.

### Phase 2 — 포트폴리오 시나리오 시스템

프로젝트별 노드 구성과 액션-결과 전이를 데이터로 선언할 수 있는 구조를 만든다.

### Phase 3 — 3D 결과 노드

Three.js 기반 3D 뷰어를 노드에 통합하고 Rhino 모델 전달 형식과 입력 이벤트 정책을 확정한다.

### Phase 4 — 콘텐츠 및 제품화

실제 프로젝트 콘텐츠, 반응형/접근성, 성능, 내비게이션, 배포 품질을 완성한다.

## Plans

### Phase 0 tasks

- [x] 목표 경험과 비계산형 고정 시나리오를 정의한다.
- [x] 요구된 색상과 기본 시각 규칙을 기록한다.
- [x] 구현 가능성, 권장 상태 모델, 주요 위험을 정리한다.
- [x] 격자 15×5단위와 이동 중 정렬 스냅 규칙을 확정한다.
- [x] 300%에서 1단위=1 CSS px, 20%~300% 줌, 우클릭 pan을 확정한다.

### Phase 1 tasks

- [x] 하나의 월드 좌표계와 viewport pan/zoom 변환을 구현한다.
- [x] 배경색과 15×5단위 격자를 viewport에 맞게 렌더링한다.
- [x] 패널, 덧셈 컴포넌트, 포트의 정상/경고 시각 상태를 구현한다.
- [x] 노드 드래그와 외곽·중앙·포트 축 정렬 스냅을 구현한다.
- [x] 출력 포트에서 입력 포트까지 임시 cubic Bézier 와이어를 구현한다.
- [x] 유효 포트 판정, 연결 확정, 취소를 구현한다.
- [x] 누락된 A 입력의 경고 표시와 hover/focus 설명 메시지를 구현한다.
- [x] `5 → A` 연결 시 경고 해제 및 결과 `8` 전이를 구현한다.
- [x] 공통 월드 변환으로 노드 이동·pan·zoom 시 포트와 와이어의 좌표 일치를 구현한다.
- [ ] 서로 다른 device pixel ratio와 브라우저 크기에서 1월드 단위의 렌더링을 검증한다.
- [x] 프로토타입 구현 결과와 후속 브라우저 QA 필요성을 기록한다.

### Phase 2 tasks

- [ ] 노드, 포트, 연결, 시나리오 전이의 데이터 스키마를 정의한다.
- [ ] 프로젝트별 초기 상태와 허용 액션을 데이터로 로드한다.
- [ ] 프로젝트 탐색, 상태 초기화, 완료 상태의 UX를 정의한다.

### Phase 3 tasks

- [ ] Rhino 모델의 웹 전달 형식과 변환 파이프라인을 결정한다.

### Phase 4 tasks
